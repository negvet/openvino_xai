class openvino_xai.common.TaskType(value)

Enum representing the different task types:

Contains the following values:

CLASSIFICATION - Classification task. DETECTION - Detection task.

class openvino_xai.common.XAIMethodType(value)

Enum representing the different XAI methods:

Contains the following values:

ACTIVATIONMAP - ActivationMap method. RECIPROCAM - ReciproCAM method. VITRECIPROCAM - VITReciproCAM method. DETCLASSPROBABILITYMAP - DetClassProbabilityMap method. RISE - RISE method.

Explanation

To explain the model (getting saliency maps), use openvino_xai.explanation

Interface for getting explanation.

class openvino_xai.explanation.ExplainMode(value)

Enum describes different explain modes.

Contains the following values:

WHITEBOX - The model is explained in white box mode, i.e. XAI branch is getting inserted into the model graph. BLACKBOX - The model is explained in black box model.

class openvino_xai.explanation.PostProcessParameters(normalize: bool = False, resize: bool = False, colormap: bool = False, overlay: bool = False, overlay_weight: float = 0.5)

PostProcessParameters parametrize postprocessing of saliency maps.

Parameters:

  • normalize (bool) – If True, normalize saliency map into [0, 255] range (filling the whole range). By default, normalization to [0, 255] range is embedded into the IR model. Therefore, normalize=False here by default.

  • resize (bool) – If True, resize saliency map to the input image size.

  • colormap (bool) – If True, apply colormap to the grayscale saliency map.

  • overlay (bool) – If True, generate overlay of the saliency map over the input image.

  • overlay_weight (float) – Weight of the saliency map when overlaying the input data with the saliency map.

class openvino_xai.explanation.PostProcessor(explanation: ExplanationResult, data: ndarray | None = None, post_processing_parameters: PostProcessParameters = PostProcessParameters(normalize=False, resize=False, colormap=False, overlay=False, overlay_weight=0.5))

PostProcessor implements post-processing for the explanation result.

Parameters:

  • explanation (ExplanationResult) – Explanation result object.

  • data (ExplanationResult) – Input data.

  • post_processing_parameters (PostProcessParameters) – Parameters that define post-processing.

postprocess() → ExplanationResult

Saliency map postprocess method. Returns ExplainResult object with processed saliency map, that can have one of SaliencyMapLayout layouts.

class openvino_xai.explanation.SaliencyMapLayout(value)

Enum describes different saliency map layouts.

Saliency map can have the following layout:

ONE_MAP_PER_IMAGE_GRAY - BHW - one map per image ONE_MAP_PER_IMAGE_COLOR - BHWC - one map per image, colormapped MULTIPLE_MAPS_PER_IMAGE_GRAY - BNHW - multiple maps per image MULTIPLE_MAPS_PER_IMAGE_COLOR - BNHWC - multiple maps per image, colormapped

class openvino_xai.explanation.TargetExplainGroup(value)

Enum describes different target explanation groups.

Contains the following values:

IMAGE - Global (single) saliency map per image. ALL - Saliency map per each possible target. PREDICTIONS - Saliency map per each prediction (prediction = target). CUSTOM - Saliency map per each specified target.

class openvino_xai.algorithms.RISE

RISEExplainer explains classification models in black-box mode using RISE (https://arxiv.org/abs/1806.07421).

classmethod run(compiled_model: Model, preprocess_fn: Callable[[ndarray], ndarray], postprocess_fn: Callable[[OVDict], ndarray], data: ndarray, explain_target_indices: List[int] | None = None, num_masks: int = 5000, num_cells: int = 8, prob: float = 0.5, seed: int = 0, normalize: bool = True)

Generates inference result of the RISE algorithm.

Parameters:

  • compiled_model (ov.Model) – Compiled model.

  • preprocess_fn (Callable[[np.ndarray], np.ndarray]) – Preprocessing function.

  • postprocess_fn (Callable[[ov.utils.data_helpers.wrappers.OVDict], np.ndarray]) – Postprocessing functions, required for black-box.

  • data (np.ndarray) – Input image.

  • explain_target_indices (List[int]) – List of target indices to explain.

  • num_masks (int) – Number of generated masks to aggregate.

  • num_cells (int) – Number of cells for low-dimensional RISE random mask that later will be up-scaled to the model input size.

  • prob (float) – With prob p, a low-res cell is set to 1; otherwise, it’s 0. Default: 0.5.

  • seed (int) – Seed for random mask generation.

  • normalize (bool) – Whether to normalize output or not.

openvino_xai.insertion.insert_xai(model: Model | str, task_type: TaskType, insertion_parameters: InsertionParameters | None = None) → Model

Function that inserts XAI branch into IR.

Usage:

model_xai = openvino_xai.insert_xai(model, task_type=TaskType.CLASSIFICATION)

Parameters:

  • model (Union[ov.Model, str]) – Original IR or path to .xml.

  • task_type (TaskType) – Type of the task.

  • insertion_parameters (InsertionParameters) – Insertion parameters that parametrize white-box method, that will be inserted into the model graph (optional).

Returns:

IR with XAI branch.
